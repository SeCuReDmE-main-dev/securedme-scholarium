# Benchmark Workspace Rules

- Use this folder to persist indexed benchmark dossier context for this codebase.
- Read local repo truth before external research.
- Prefer official docs, pricing pages, standards, and maintainer material over commentary.
- Keep major conclusions labeled by evidence quality.
- Keep root files limited to workspace-wide guidance, memory, pointers, and registries.
- Store each benchmark entry under `benchmarks/<slug>/`.
- Require a benchmark-validity check before treating a benchmark as accepted.
- Store generated harnesses and measured results inside each benchmark entry.
- Preserve v1 flat benchmark artifacts as migration input instead of deleting them automatically.
