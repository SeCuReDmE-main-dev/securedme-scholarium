# Scholarium Visual Identity Execution Brief

Status: accepted refinement of actions 133, 137, 138 and 139. Execution is
scheduled in Group 12, after the product and social foundations around action
100.

## Source of truth

The official working source is the repository `assets/` directory. The current
inventory contains 49 PNG images, one MP4 video and one README across banners,
badges, icon concepts, logo concepts and SecuredMe Education assets. Numeric
filenames such as `1.png` are working identifiers, not a reason to discard or
replace the user's design work.

## Required execution

1. Preserve every original asset and its relative path during intake.
2. Build a visual contact sheet and record dimensions, aspect ratio, file size,
   transparency, dominant colors, intended role and duplicate similarity.
3. Create a semantic asset manifest mapping each original path to a proposed
   product name, usage, light/dark context and approval state.
4. Select the Scholarium and Scholarium Teach identity system from the existing
   work before generating or sourcing replacements.
5. Replace the remaining starter-template visual language on the landing page
   and authenticated application with one coherent system.
6. Apply the identity to navigation, real product screens, charts, timelines,
   strength maps, evidence views, empty states and responsive controls.
7. Optimize approved derivatives for the web while keeping archival originals
   untouched.
8. Validate desktop, tablet and mobile visually with Playwright, including
   image rendering, cropping, contrast, focus, text fit and zero overlap.

## Action mapping

- **133:** align the branded navigation with the accepted product sitemap.
- **137:** audit, name and apply the visual identity from `assets/`.
- **138:** encode the identity in stable accessible components and tokens.
- **139:** use the identity in charts, timelines, strengths and evidence views.

## Completion boundary

This refinement is not complete when files are merely renamed. Completion
requires an approved manifest, implemented product surfaces and direct browser
proof. Original numeric files remain available until the manifest and visual
selection have been validated.
